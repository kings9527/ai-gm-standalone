## 安装指南

### Windows

1. 下载最新版本安装包：`AI-GM-Standalone-Setup-1.0.0.exe`
2. 双击运行安装程序，按向导提示完成安装
3. 安装完成后，桌面会出现「AI-GM Standalone」快捷方式

> **截图占位**：`[SCREENSHOT: windows-installer-wizard]` — Windows 安装向导界面

### macOS

1. 下载 `AI-GM-Standalone-1.0.0.dmg`
2. 双击挂载 DMG，将应用图标拖拽到「应用程序」文件夹
3. 首次运行时若提示「无法验证开发者」，前往 **系统设置 → 隐私与安全性** → 点击「仍要打开」

> **截图占位**：`[SCREENSHOT: macos-dmg-install]` — macOS DMG 安装界面

### Linux

**方式一：AppImage（推荐）**

```bash
chmod +x AI-GM-Standalone-1.0.0.AppImage
./AI-GM-Standalone-1.0.0.AppImage
```

**方式二：deb 包（Debian/Ubuntu）**

```bash
sudo dpkg -i ai-gm-standalone_1.0.0_amd64.deb
sudo apt-get install -f  # 修复依赖
```

> **截图占位**：`[SCREENSHOT: linux-appimage-run]` — Linux 终端运行 AppImage

## 首次启动

首次启动时，应用会自动完成以下初始化：

1. 启动后端服务（端口 9742）
2. 创建用户数据目录（`~/AI-GM/` 或系统对应的用户数据目录）
3. 初始化 SQLite 数据库（`ai-gm.db`）

启动完成后，主界面如下：

## 模组导入与管理

### 导入模组

**从文件导入：**

1. 进入 **模组管理** 页面
2. 点击「导入模组」按钮
3. 选择 `.json` 格式的模组文件
4. 导入成功后，模组会出现在列表中

> **截图占位**：`[SCREENSHOT: module-import-dialog]` — 文件选择对话框与导入成功提示

**模组文件格式要求：**

模组文件为 JSON 格式，包含 `id`、`name`、`system`、`scenes`、`npcs` 等字段。具体格式规范参见 [开发者指南](developer-guide.md#模组格式规范)。

### 管理模组

在模组管理页面，你可以：

- **查看** 已导入的模组列表（名称、作者、系统、版本）
- **导出** 模组为 JSON 文件（备份或分享）
- **删除** 不再需要的模组

> **截图占位**：`[SCREENSHOT: module-manager-list]` — 模组管理页面，显示多个模组卡片

## 开始游戏

### 进入游戏

1. 确保已导入至少一个模组
2. 点击首页「继续游戏」或直接访问 `#/play`
3. 应用会加载默认演示模组（或上次游玩的模组）

> **截图占位**：`[SCREENSHOT: game-loading-screen]` — 加载界面，带旋转加载动画与骨架屏

### 游戏界面

游戏界面采用视觉小说风格：

- **顶部**：场景标题（可选）
- **中央**：背景图 + 角色立绘（Sprite）
- **底部**：对话文本框（打字机效果）
- **右下角**：选项按钮 / 操作按钮

> **截图占位**：`[SCREENSHOT: gameplay-main-view]` — 典型游戏场景，包含背景、角色立绘和对话

### 操作方式

| 操作 | 说明 |
|------|------|
| 点击对话文本 | 加速打字机效果 / 跳至下一句 |
| 点击选项 | 选择剧情分支 |
| 点击立绘区域 | 部分场景可触发交互 |
| 按 `ESC` / 点击菜单按钮 | 打开游戏内菜单（存档、读档、设置、退出） |

> **录屏占位**：`[VIDEO: gameplay-interaction-demo]` — 演示对话推进、选项选择、场景切换

## 战斗系统

当剧情进入战斗场景时，屏幕会切换为战斗界面：

### 战斗界面元素

- **顶部 HUD**：回合数、当前行动者
- **中央**：战斗实体（玩家、敌人、盟友）及其 HP 条
- **底部**：行动菜单（攻击、技能、物品、防御、逃跑）
- **右侧**：战斗日志（实时滚动）

> **截图占位**：`[SCREENSHOT: combat-interface]` — 战斗界面，显示多个敌人与行动菜单

### 行动类型

| 行动 | 说明 |
|------|------|
| **攻击** | 使用基础武器（徒手/枪械）进行 d100 命中检定 |
| **技能** | 消耗 MP 或 SAN 使用特殊技能 |
| **物品** | 使用背包中的消耗品 |
| **防御** | 提升本回合防御力 |
| **逃跑** | 敏捷检定，成功后脱离战斗 |

### 战斗结果

- **大成功**（骰出 ≤ 5）：双倍伤害
- **成功**（骰出 ≤ 目标值）：正常伤害
- **失败**（骰出 > 目标值）：未命中
- **大失败**（骰出 ≥ 96）：可能反伤自己

> **录屏占位**：`[VIDEO: combat-turn-demo]` — 演示一回合完整流程：选择行动 → 命中检定 → 造成伤害 → 日志更新

### 战斗结束

- **胜利**：所有敌人被消灭，回到剧情场景
- **失败**：玩家 HP 归零，触发结局或 Game Over
- **逃跑**：成功逃离战斗，回到剧情场景

## 存档与读档

### 手动存档

1. 在游戏中按 `ESC` 或点击菜单按钮
2. 选择「存档」
3. 选择一个空槽位或覆盖已有存档
4. 可选：输入存档名称

> **截图占位**：`[SCREENSHOT: save-panel-ui]` — 存档面板，显示多个存档槽位与缩略图

### 自动存档

游戏在以下时刻会自动保存（覆盖自动存档槽）：

- 场景切换时
- 战斗开始前
- 关键剧情节点

### 读档

1. 在游戏内菜单选择「读档」
2. 从存档列表中选择要读取的存档
3. 确认后，游戏将恢复到存档时的场景和状态

> **截图占位**：`[SCREENSHOT: load-panel-ui]` — 读档面板，高亮显示自动存档与手动存档

### 存档位置

存档数据存储在本地数据库中，同时包含：

- 玩家状态（HP、SAN、属性、背包）
- 当前场景与历史记录
- 全局变量与 NPC 状态
- 视觉小说引擎的快照（背景、立绘位置、对话进度）
- 存档缩略图（截图）

## 设置

### LLM 配置

配置 AI 对话引擎，用于模组生成、风格分析、剧情推演：

| 提供商 | 说明 | 需要配置 |
|--------|------|----------|
| **OpenAI** | GPT-4o / GPT-4o-mini | API Key |
| **Claude** | Claude 3.5 Sonnet | API Key |
| **Ollama** | 本地模型（如 Llama 3.2） | 本地服务地址 |

> **截图占位**：`[SCREENSHOT: settings-llm-panel]` — LLM 设置面板，显示提供商选择与 Key 输入框

### 图片配置

- **Unsplash API Key**：配置后可用真实图片搜索（否则使用 Picsum 随机图）
- **DALL-E Key**：配置后可 AI 生成图片
- **默认策略**：搜索 / 生成 / 上传

### 游戏设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 打字机速度 | 每个字符显示间隔（毫秒） | 30ms |
| 字体大小 | 对话文本字体大小 | 16px |
| 自动前进延迟 | 对话结束后自动推进的等待时间 | 0（关闭） |
| 允许跳过未读 | 是否可以跳过未读过的文本 | 否 |

### 主题设置

- **深色模式 / 浅色模式 / 自动跟随系统**
- 自定义 CSS 变量（高级）

> **截图占位**：`[SCREENSHOT: settings-theme-toggle]` — 主题切换演示，深色与浅色模式对比

## 图片管理

图片管理页面用于管理游戏中的图片资源：

### 搜索图片

1. 进入 **图片管理** 页面
2. 输入关键词（如「dark forest」「cyberpunk city」）
3. 选择图片类型（背景 / 立绘 / 肖像）
4. 点击「搜索」

> **截图占位**：`[SCREENSHOT: image-search-results]` — 图片搜索结果网格，显示多张缩略图

### 生成图片（AI）

1. 输入描述性提示词（Prompt）
2. 选择图片类型（背景尺寸为 1792×1024，立绘为 1024×1792）
3. 点击「生成」
4. 生成完成后自动保存到本地

> **截图占位**：`[SCREENSHOT: image-generation-panel]` — AI 图片生成界面，显示 Prompt 输入框与生成结果

### 上传图片

支持从本地文件上传：
- 格式：JPG、PNG、GIF、WebP、BMP
- 最大限制：50MB（通过 Base64 编码传输）

> **截图占位**：`[SCREENSHOT: image-upload-dialog]` — 本地文件选择对话框

## AI 风格分析器

风格分析器用于从故事文本中提取视觉风格特征，生成主题配置：

### 使用步骤

1. 进入 **AI 风格分析器** 页面
2. 粘贴或输入故事文本（至少 200 字）
3. 选择目标规则系统（COC / D&D 5e / 自定义）
4. 点击「分析」
5. 等待 AI 返回分析结果：
   - 色彩方案（背景色、强调色、文本色）
   - 氛围与时代感
   - 艺术风格与光照
   - 情绪关键词
   - 推荐字体与特效

> **截图占位**：`[SCREENSHOT: style-analyzer-input]` — 风格分析器输入界面，显示文本框与分析按钮

### 应用风格

分析完成后，你可以：
- 将风格配置应用到当前模组
- 保存为自定义风格模板
- 导出为 JSON 文件

> **截图占位**：`[SCREENSHOT: style-analyzer-result]` — 分析结果展示，包含色板预览与风格标签

## 模组生成器

模组生成器允许你通过 AI 将文本故事转换为完整可玩的模组：

### 使用步骤

1. 进入 **上传故事，创建模组** 页面
2. 上传故事文件（TXT / Markdown）或直接在文本框中粘贴
3. 选择规则系统（COC / D&D 5e / 自定义）
4. 点击「生成模组」
5. AI 会自动：
   - 提取场景结构
   - 生成 NPC 角色
   - 设计对话与选项
   - 配置战斗（如有）
   - 推荐图片关键词

> **截图占位**：`[SCREENSHOT: generator-upload-page]` — 模组生成器上传页面，显示文件拖拽区域

### 预览与编辑

生成完成后，你可以预览模组结构：
- 场景列表与连接关系
- NPC 属性与立绘
- 对话树与分支选项

确认无误后，点击「保存到模组库」即可导入。

> **截图占位**：`[SCREENSHOT: generator-preview-panel]` — 模组预览面板，显示场景流程图

## 故障排除

### 无法启动应用

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 启动后白屏 | 后端服务启动失败 | 检查端口 9742 是否被占用；重启应用 |
| 提示「后端服务启动失败」 | Node.js 环境问题 | 确保系统已安装 Node.js 20+；重新安装应用 |
| macOS 提示无法验证开发者 | 签名问题 | 系统设置 → 隐私与安全性 → 仍要打开 |

### 模组加载失败

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 导入后模组不显示 | JSON 格式错误 | 使用 JSON 校验工具检查文件；参考模组格式规范 |
| 游玩时场景卡住 | 场景 `start_scene` 指向不存在的场景 | 检查模组 JSON 中 `start_scene` 字段是否有效 |
| 图片无法加载 | 图片路径错误 | 确保图片 URL 可访问；或改用本地图片上传 |

### LLM 连接问题

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| AI 对话无响应 | API Key 未配置或错误 | 检查设置中的 LLM 配置；确认 Key 有效 |
| 流式输出卡顿 | 网络延迟 | 切换至非流式模式；或更换更快的模型（如 gpt-4o-mini） |
| Ollama 连接失败 | 本地服务未启动 | 确保 Ollama 已安装并运行（`ollama serve`） |

### 存档问题

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 读档后场景错乱 | 模组版本变更 | 确保模组未被修改；重新创建存档 |
| 存档丢失 | 用户数据目录被清理 | 检查 `~/AI-GM/` 目录是否存在；定期导出备份 |

### 性能问题

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 动画卡顿 | 图片过大 | 使用压缩后的图片；减少同时显示的立绘数量 |
| 内存占用高 | 长时间运行后缓存堆积 | 重启应用；清理图片缓存 |

## 附录

### 快捷键

| 快捷键 | 功能 |
|--------|------|
| `ESC` | 打开/关闭游戏内菜单 |
| `Space` | 推进对话（同点击） |
| `Ctrl + S` | 快速存档（游戏中） |
| `Ctrl + L` | 快速读档（游戏中） |
| `F11` | 全屏切换 |
| `F12` | 开发者工具（仅开发模式） |

### 用户数据目录

| 平台 | 路径 |
|------|------|
| Windows | `%APPDATA%\AI-GM Standalone` 或 `C:\Users\<用户名>\AI-GM` |
| macOS | `~/Library/Application Support/AI-GM Standalone` 或 `~/AI-GM` |
| Linux | `~/.config/AI-GM Standalone` 或 `~/AI-GM` |

### 日志文件

日志保存在用户数据目录下的 `logs/` 文件夹中，包含：
- 主进程日志（`main.log`）
- 后端服务日志（`backend.log`）
- 渲染进程日志（`renderer.log`）

遇到问题时，请附上相关日志文件提交反馈。
